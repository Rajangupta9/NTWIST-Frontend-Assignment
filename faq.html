What is responsive web design?

Responsive web design is an approach to web design that makes web pages render well on a variety of devices and window or screen sizes. It uses HTML and CSS to automatically resize, hide, shrink, or enlarge a website to make it look good on all devices (desktops, tablets, and phones).

Key components include flexible grids, flexible images, and media queries in CSS.

What are the differences between HTML and HTML5?

HTML5 introduced several new features over HTML:

  • Semantic elements like <header>, <footer>, <article>, <section>, <nav>
  • New form input types (date, email, time, etc.)
  • Graphics elements: <canvas> and <svg>
  • Multimedia elements: <audio> and <video>
  • Better storage options with localStorage and sessionStorage
What is the difference between CSS Grid and Flexbox?

CSS Grid and Flexbox are both layout models in CSS but serve different purposes:

CSS Grid is designed for two-dimensional layouts (rows AND columns). It's ideal for page layouts and complex grid-based interfaces.

Flexbox is designed for one-dimensional layouts (either rows OR columns). It's perfect for components, navigation bars, and smaller-scale layouts.

They can be used together - Grid for the overall page layout and Flexbox for the components within that layout.

Event bubbling and capturing are two ways of event propagation in the HTML DOM:

Event Bubbling: When an event occurs on an element, it first runs the handlers on it, then on its parent, then all the way up. The event "bubbles up" from the innermost element to the outermost one.

Event Capturing: It's the opposite of bubbling. The event is first captured by the outermost element and propagates to the target element.

Most browsers use bubbling by default. Capturing can be enabled by setting the third parameter of addEventListener to true.

Browser rendering involves several steps:

  1. DOM Construction: Parse HTML to create the Document Object Model
  2. CSSOM Construction: Parse CSS to create the CSS Object Model
  3. Render Tree Construction: Combine DOM and CSSOM
  4. Layout: Calculate the size and position of each element
  5. Paint: Fill in pixels with colors, images, etc.
  6. Compositing: Draw layers in the correct order

Understanding this process helps optimize website performance.

The Virtual DOM is a programming concept where a virtual representation of the UI is kept in memory and synced with the "real" DOM. This process is called reconciliation.

When state changes in applications like React:

  1. A new virtual DOM is created representing the updated UI
  2. The new virtual DOM is compared with the previous one (diffing)
  3. Only the differences are updated in the real DOM

This approach minimizes direct DOM manipulation and improves performance, especially in complex web applications.
